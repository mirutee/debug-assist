 Próximos passos para produção

  1. Supabase (banco de dados)
  - Criar conta em supabase.com e novo projeto
  - Rodar o SQL da tabela diagnosticos (está no CLAUDE.md)
  - Copiar SUPABASE_URL e SUPABASE_KEY para o .env

  2. Deploy
  - Criar conta no https://railway.app ou https://render.com (gratuitos)
  - Conectar o repositório GitHub
  - Adicionar as variáveis de ambiente: PORT, API_KEY, SUPABASE_URL, SUPABASE_KEY
  - Deploy automático em 1 clique

  3. Domínio e API Key real
  - Gerar uma API_KEY segura (ex: openssl rand -hex 32)
  - Configurar domínio próprio (ex: api.devinsight.com)

  4. Publicar o SDK
  - Criar conta no npmjs.com
  - Rodar npm publish dentro da pasta sdk/
  - Usuários instalam com npm install devinsight-sdk

  5. Validação de mercado
  - Publicar landing page (o copy já está no Prompt.txt)
  - Postar em r/webdev, LinkedIn, grupos de dev
  - Coletar os primeiros 10 usuários via lista de espera

  6. Opcional (v2)
  - Integrar OpenAI para diagnósticos mais inteligentes
  - Criar dashboard web para visualizar os logs do Supabase
  - Adicionar planos com rate limit por API Key (Free/Pro/Scale)
